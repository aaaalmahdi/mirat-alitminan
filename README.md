# مرآة الاطمئنان — V6 Readiness

تطبيق ويب ثابت، عربي أولاً، لخدمة توعوية في الكشف المبكر عن الأمراض المزمنة. يحدد باقة توجيهية أولية حسب **العمر الحقيقي والجنس**، مع تجربة اختيارية وغير طبية لتقدير العمر الظاهري.

> هذا الإصدار تجريبي. المحتوى الطبي قيد المراجعة والاعتماد الرسمي، ولا يجوز إضافة شعار الجهة الرسمي أو وصف الباقات بأنها معتمدة نهائياً قبل استكمال نموذج الاعتماد.

## الروابط

- المرآة: `https://aaaalmahdi.github.io/mirat-alitminan/`
- معلومات الخدمة والباقات: `https://aaaalmahdi.github.io/mirat-alitminan/info.html`
- صفحة الباركودين: `https://aaaalmahdi.github.io/mirat-alitminan/qr.html`

لا تستخدم رابط الجذر `https://aaaalmahdi.github.io` لأنه ليس مستودع هذا المشروع.

## ما تغير في V6

- أزيلت نتيجة العمر العشوائية بالكامل. إذا تعذر النموذج، يتابع المستخدم بدون كاميرا.
- أصبح `face-api.js` يُحمّل بعد موافقة المستخدم واختياره لمسار الكاميرا فقط.
- أضيف نموذج محاذاة ملامح الوجه قبل تقدير العمر لزيادة الثبات.
- أضيف تحقق صريح ومنفصل للعمر والجنس، دون افتراض الجنس.
- استبدلت عبارة «الفحوصات المطلوبة» بعبارات توجيهية آمنة مثل «الفحوصات المقترحة».
- أصبحت الكاميرا خياراً ثانوياً، والمسار المباشر هو الخيار الأساسي.
- أضيفت إدارة تركيز قارئ الشاشة، ورسائل خطأ معلنة، ودعم التكبير والحركة المخفضة.
- أضيف وضعان:
  - الوضع الافتراضي للهواتف: بلا إعادة تشغيل تلقائية.
  - وضع البوث: `?mode=kiosk` مع تنبيه قبل إعادة الجلسة.
- أصبحت الإحصاءات مفصولة إلى يومية وتراكمية، ولم يعد فتح صفحة المعلومات يُجمع مرتين.
- لوحة الإحصاءات لا تتاح إلا عند فتح الرابط مع `?admin=1` ثم النقر على منطقة الإدارة خمس مرات.
- استبدل الرمز شبه الرسمي بعلامة تجريبية محايدة.
- أضيفت صفحة معلومات ثنائية اللغة، وحالة اعتماد ظاهرة، وقسم خصوصية.
- أضيف باركودان ثابتان: واحد للمرآة، وآخر للمعلومات والباقات.
- أصبح GitHub Actions ينشر ملفات الموقع العامة فقط من `_site` بعد نجاح الاختبارات.
- أضيفت اختبارات آلية لتغطية الأعمار 18–75، وتطابق الإصدارات، والصياغة الآمنة، والأصول، ومسار النشر.

## الخصوصية

- لا يحفظ التطبيق صوراً أو فيديو.
- لا يرفع التطبيق صورة الكاميرا إلى خادم تابع للمشروع.
- معالجة الكاميرا تتم داخل جهاز المستخدم أثناء الجلسة.
- الأسئلة الاختيارية لا تحفظ في الإحصاءات.
- الإحصاءات محلية ومجمعة على الجهاز، ولا تشمل أسماء أو صوراً أو أرقاماً شخصية أو أعماراً فردية.
- قد يسجل مزود الاستضافة بيانات تشغيلية تقنية اعتيادية وفق سياسته.

## الباقات والقواعد الطبية

مصدر البيانات الرئيسي هو:

```text
packages.json
```

النسخة الاحتياطية المحلية المولدة هي:

```text
packages-data.js
```

لا تعدل `packages-data.js` يدوياً. بعد تعديل `packages.json` شغّل:

```bash
npm run sync:packages
npm test
```

حالة المحتوى الحالية محفوظة في `packages.json` داخل `metadata`، وهي:

```text
Pending official review and approval
```

استخدم `CLINICAL_APPROVAL.md` لتوثيق المالك الطبي، والمراجع، والمصدر، وتاريخ الاعتماد والمراجعة القادمة.

## الشعار الرسمي

الملف الحالي:

```text
assets/brand/mirat-mark.svg
```

علامة تجريبية محايدة، وليست شعار مملكة البحرين أو شعار مراكز الرعاية الصحية الأولية.

عند استلام الشعار الرسمي:

1. أضف ملف SVG الأصلي أو PNG شفافاً عالي الدقة إلى `assets/brand/`.
2. استبدل مراجع العلامة في `index.html` و`info.html` و`qr.html`.
3. التزم بالمساحة الآمنة والألوان والحد الأدنى للحجم.
4. أزل عبارة النسخة التجريبية فقط بعد اعتماد المحتوى والهوية كتابياً.

## الباركودان

- `assets/qr/mirror-qr.svg`
- `assets/qr/info-packages-qr.svg`

الروابط داخل الباركودين ثابتة ولا تحتوي على `?v=`. كسر الكاش يتم في ملفات CSS وJavaScript فقط.

## التشغيل المحلي

يجب تشغيل الموقع عبر خادم HTTP، لا بفتح `index.html` مباشرة:

```bash
python3 -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080/
```

الكاميرا على الأجهزة الأخرى تحتاج HTTPS.

## التحقق

لا توجد تبعيات تشغيلية مطلوبة للاختبارات:

```bash
npm test
```

الاختبار يتحقق من:

- تطابق `packages.json` و`packages-data.js`.
- تطابق رقم الإصدار في الملفات.
- وجود باقة واحدة فقط لكل عمر من 18 إلى 75 للذكر والأنثى.
- عدم ظهور باقة خارج النطاق.
- غياب العمر العشوائي.
- تحميل مكتبة الكاميرا بعد الموافقة فقط.
- غياب `maximum-scale=1`.
- الصياغة الطبية الآمنة.
- وجود ملفات الهوية والباركود.
- نشر `_site` فقط.

## النشر

المسار الوحيد المعتمد للمشروع:

```text
Settings → Pages → Source: GitHub Actions
```

الملف:

```text
.github/workflows/pages.yml
```

مجلد `docs/` تاريخي وغير مستخدم في النشر. لا تضبط Pages على `main/docs`.

راجع `DEPLOYMENT_CHECKLIST.md` قبل الدمج أو الطباعة أو الاستخدام الميداني.
