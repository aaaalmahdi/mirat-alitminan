# مرآة الاطمئنان — V7 Identity and Package Approval

تطبيق ويب ثابت، عربي أولاً، لخدمة توعوية في الكشف المبكر عن الأمراض المزمنة. يحدد باقة توجيهية أولية حسب **العمر الحقيقي والجنس**، مع تجربة اختيارية وغير طبية لتقدير العمر الظاهري.

> تم اعتماد جدول الباقات وقواعد الفحوصات الشرطية الحالية كما هي للاستخدام في المشروع بتاريخ 14 يوليو 2026، وأضيف الشعار الذي قدمه مالك المشروع. يظل الاستحقاق النهائي لكل فحص مرتبطاً بالملف الإلكتروني الصحي ويؤكده الموظف المختص.

هذا الاعتماد هو اعتماد مالك المشروع للجدول المقدم، ولا يدّعي وجود مراجعة حوكمة سريرية خارجية منفصلة.

## الروابط

- المرآة: `https://aaaalmahdi.github.io/mirat-alitminan/`
- معلومات الخدمة والباقات: `https://aaaalmahdi.github.io/mirat-alitminan/info.html`
- صفحة الباركودين: `https://aaaalmahdi.github.io/mirat-alitminan/qr.html`

لا تستخدم رابط الجذر `https://aaaalmahdi.github.io` لأنه ليس مستودع هذا المشروع.

## ما تغير في V7

- أضيف الشعار المقدم للمشروع في `assets/brand/primary-healthcare-centres-emblem.svg`.
- استبدلت العلامة التجريبية في المرآة وصفحة المعلومات وصفحة طباعة الباركودين.
- أزيلت عبارات انتظار الهوية والمراجعة من الصفحات العامة.
- سُجل اعتماد جدول الباقات وقواعد الفحوصات الشرطية الحالية **دون أي تعديل**.
- بقيت صياغة التوعية والتوجيه الأولي والتحقق من الاستحقاق محفوظة.
- أضيفت طبقة اعتماد مستقلة حتى تبقى بيانات الباقات الأصلية دون تغيير:
  - `approval-data.js`
  - `app-approval.js`
  - `info-approval.js`
  - `official-brand.css`
- وثق نطاق القرار وحدوده في `CLINICAL_APPROVAL.md`.
- بقي إصدار تكوين الباقات `20260714-v6`، بينما إصدار الواجهة والهوية هو `20260714-v7`.

## تحسينات V6 المحتفظ بها

- أزيلت نتيجة العمر العشوائية بالكامل؛ إذا تعذر النموذج يتابع المستخدم بدون كاميرا.
- يُحمّل `face-api.js` بعد موافقة المستخدم واختياره لمسار الكاميرا فقط.
- أضيف نموذج محاذاة ملامح الوجه قبل تقدير العمر لزيادة الثبات.
- يوجد تحقق صريح ومنفصل للعمر والجنس، دون افتراض الجنس.
- تستخدم الواجهة عبارات آمنة مثل «الفحوصات المقترحة».
- المسار المباشر هو الخيار الأساسي، والكاميرا خيار ثانوي اختياري.
- توجد إدارة تركيز لقارئ الشاشة، ورسائل خطأ معلنة، ودعم للتكبير والحركة المخفضة.
- الوضع الافتراضي للهواتف بلا إعادة تشغيل تلقائية.
- وضع البوث: `?mode=kiosk` مع تنبيه قبل إعادة الجلسة.
- الإحصاءات يومية وتراكمية ومحلية فقط.
- لوحة الإحصاءات تتاح عبر `?admin=1` ثم خمس نقرات على منطقة الإدارة.
- GitHub Actions ينشر ملفات الموقع العامة فقط من `_site` بعد نجاح الاختبارات.

## الخصوصية

- لا يحفظ التطبيق صوراً أو فيديو.
- لا يرفع التطبيق صورة الكاميرا إلى خادم تابع للمشروع.
- معالجة الكاميرا تتم داخل جهاز المستخدم أثناء الجلسة.
- الأسئلة الاختيارية لا تحفظ في الإحصاءات.
- الإحصاءات محلية ومجمعة على الجهاز، ولا تشمل أسماء أو صوراً أو أرقاماً شخصية أو أعماراً فردية.
- قد يسجل مزود الاستضافة بيانات تشغيلية تقنية اعتيادية وفق سياسته.

## الباقات والقواعد الطبية

مصدر البيانات الرئيسي:

```text
packages.json
```

النسخة الاحتياطية المحلية المولدة:

```text
packages-data.js
```

تم إبقاء الملفين وقواعدهما كما كانا في إصدار V6. لا تعدل `packages-data.js` يدوياً. بعد أي تعديل مستقبلي في `packages.json` شغّل:

```bash
npm run sync:packages
npm test
```

الباقات المعتمدة للاستخدام في المشروع كما هي:

| Code | Package | Age | Gender |
|---|---|---:|---|
| P1 | باقة الشباب | 18–39 | General |
| P2 | باقة الأربعينات 1 | 40–44 | General |
| P3 | باقة الأربعينات 2 | 45–49 | General |
| M1 | باقة صحة الرجل | 50–65 | Male |
| F1 | باقة صحة المرأة | 50–65 | Female |
| G1 | باقة العمر الذهبي | 66–75 | General |

قواعد الفحوصات الشرطية المعتمدة كما هي:

- Female 30–65: Cervical smear.
- Female 40–75: Mammogram.
- General 45–75: Diabetes screening.
- General 45–75: Colon screening.
- Male 50–70: Prostate screening.

## حدود النتيجة

التطبيق يقدم توعية وتوجيهاً أولياً فقط. لا يحدد:

- دورية الفحص.
- أثر النتائج السابقة.
- الاستثناءات الفردية.
- الاستحقاق النهائي.
- التشخيص أو الخطة العلاجية.

يؤكد الموظف المختص الفحوصات المستحقة باستخدام الملف الإلكتروني الصحي وتاريخ ونتائج الفحوصات السابقة.

## الهوية المستخدمة

الملف النشط:

```text
assets/brand/primary-healthcare-centres-emblem.svg
```

يتضمن نسخة محسنة للويب من الصورة الشفافة التي قدمها مالك المشروع بتاريخ 14 يوليو 2026، مع الحفاظ على النسب والتصميم. يظهر في:

- `index.html`
- `info.html`
- `qr.html`

العلامة السابقة `assets/brand/mirat-mark.svg` محفوظة للأرشفة والرجوع فقط، ولا تستخدم في الصفحات العامة.

يجب الحفاظ على تناسب الشعار ومساحته الواضحة، وعدم وضعه فوق وحدات الباركود.

## الباركودان

- `assets/qr/mirror-qr.svg`
- `assets/qr/info-packages-qr.svg`

الروابط داخل الباركودين ثابتة ولا تحتوي على `?v=`. كسر الكاش يتم في ملفات CSS وJavaScript فقط.

## التشغيل المحلي

شغّل الموقع عبر خادم HTTP:

```bash
python3 -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080/
```

الكاميرا على الأجهزة الأخرى تحتاج HTTPS.

## التحقق

```bash
npm test
```

الاختبار يتحقق من:

- تطابق `packages.json` و`packages-data.js`.
- وجود باقة واحدة فقط لكل عمر من 18 إلى 75 للذكر والأنثى.
- عدم ظهور باقة خارج النطاق.
- غياب العمر العشوائي.
- تحميل مكتبة الكاميرا بعد الموافقة فقط.
- الصياغة الطبية الآمنة.
- وجود الشعار والباركودين.
- تطابق إصدار V7 مع طبقة الاعتماد مع بقاء تكوين الباقات V6 دون تعديل.
- نشر `_site` فقط.
